# 目前進度 2026-06-08

## 已做到的成果

- TSS3-lite 紅燈 prepare 邏輯可以讀取紅燈偵測狀態，並依階段提出放油門、輕剎、較強剎車的要求。
- UI 已能顯示車輛目前停車意圖、STOP LINE 資訊與剎車力道色條。
- qlog debug marker 已加入，方便事後看當時為什麼有沒有剎車。
- 速限控制來源已修正：優先使用 sunnypilot UI 同一套 speed limit resolver/live map，而不是只看自訂 OSM context。
- 本地已實作低負載路口距離 sidecar。
- qlog replay 工具已能把每次距離計算耗時寫入表格。

## 最新本地版本

最新距離來源改成 GPS + 本地 OSM 事件地圖：

1. 只在啟動或檔案更新時讀取 `/data/sienna_route/osm_events.geojson`。
2. 從 OSM GeoJSON 抽出紅綠燈、stop、give-way、行人穿越道、路口事件。
3. 建立小型 grid index，行進中只查附近格子。
4. 有 route 時，把候選 OSM 事件投影到 route 上，找最近的前方事件。
5. 沒 route 時，用 GPS heading cone 找前方疑似同一路徑上的事件。
6. 只有找到可信距離時，才輸出 `next_osm_event.distance_m` 給 planner。

`/intersection_distance` bridge 還保留，但定位是備援、手動驗證或外部 companion 測試入口。

## 實測觀察

- 車子有時偵測到紅燈，但只做很弱的制動，接近路口才開始不夠用的剎車。
- 有些停車可能不是我們的紅燈邏輯造成，而是 openpilot model、lead 或原本 longitudinal 行為造成。
- 沒有可靠距離時，FAR/MID/NEAR 只是一種影像級別，不是物理距離。
- 超線大約可能是一個車頭到半個車身。
- 要舒適停住，必須讓 GPS/OSM 距離提早啟動 prepare，重剎只留給臨時變燈或晚看到紅燈。
- C3X 負載必須非常保守，所以避免啟動完整 OSM sidecar。

## 目前缺口

- C3X 上還沒有驗證過真實 `/data/sienna_route/osm_events.geojson`。
- 最新本地 GPS+OSM 距離 sidecar 還沒同步到 C3X。
- qlog replay 可以驗證邏輯與耗時，但不能證明 Toyota/ACC 最終真的照做。
- 單靠影像紅燈偵測還不能可靠知道停止線物理距離。
