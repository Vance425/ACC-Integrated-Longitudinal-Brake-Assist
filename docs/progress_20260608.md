# Progress as of 2026-06-08

## What Is Working

- TSS3-lite red-light prepare logic can read red-light sidecar/debug state and request gentle coast/braking phases.
- UI overlay can show vehicle stop intent, STOP LINE information, and brake-force bar without fully covering the road view.
- qlog markers and debug params exist for later analysis of prepare/braking decisions.
- Speed-limit cap source was corrected to prefer sunnypilot speed-limit resolver/live map sources instead of only the custom OSM context file.
- A low-load intersection-distance sidecar was implemented locally.
- qlog replay tooling now measures distance-generation cost per row.

## Latest Local Implementation

The newest local distance source is GPS + local OSM event map:

1. Load `/data/sienna_route/osm_events.geojson` only at startup or when mtime changes.
2. Extract OSM traffic signals, stop signs, give-way points, crossings, and intersections.
3. Build a small grid index so runtime queries only check nearby cells.
4. If a route is active, project candidate OSM events onto the route and choose the nearest event ahead.
5. If no route is active, use current GPS heading and a forward cone to find likely same-road events.
6. Output planner-readable `next_osm_event.distance_m` only when a match is active.

The bridge endpoint `/intersection_distance` remains useful for manual or companion-provided distances, but it is not the primary design.

## Road-Test Findings So Far

- The vehicle sometimes detects red lights but only applies weak braking until too close.
- Some stops happened from openpilot/model/lead behavior rather than our intended red-light distance logic.
- Overshoot can happen by roughly a car head to half a car length when distance is unavailable or late.
- GPS/OSM distance is needed to start gentle slowdown earlier and leave heavy braking only for late-reveal or sudden-change cases.
- Load must stay low on C3X; full OSM sidecar autostart is avoided.

## Current Gaps

- No real `/data/sienna_route/osm_events.geojson` has been validated on C3X yet.
- The latest local GPS+OSM distance sidecar has not been synced to C3X because the device was offline.
- qlog replay can measure compute time and simulate distance logic, but it cannot prove Toyota/ACC final actuation.
- Camera-only red-light detection cannot reliably infer true physical stop-line distance without additional map/line context.
